# Repository Documentation

This folder contains contributor-focused documentation for the portfolio repository. It explains what the project contains, how the site is structured, where assets and content live, and which known content issues should be addressed in future cleanups.

## Documents

- [repo-overview.md](./repo-overview.md): High-level project overview, stack summary, and top-level file map.
- [site-structure.md](./site-structure.md): Page section map, anchor flow, and navigation behavior.
- [assets-map.md](./assets-map.md): CSS, JS, image, and external dependency inventory with file roles.
- [content-inventory.md](./content-inventory.md): Catalog of publications, experience, projects, skills, certifications, and contact data.
- [known-issues.md](./known-issues.md): Observed typos, encoding artifacts, and consistency issues with fix guidance.

## How To Use This Docs Folder

- Start with [repo-overview.md](./repo-overview.md) to understand the repo at a glance.
- Use [site-structure.md](./site-structure.md) and [content-inventory.md](./content-inventory.md) when updating `index.html` content.
- Use [assets-map.md](./assets-map.md) before changing styles or scripts.
- Check [known-issues.md](./known-issues.md) for content cleanups that are safe to apply in future commits.
